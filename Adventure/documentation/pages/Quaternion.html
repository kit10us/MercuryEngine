Quaternion

A quaternion represents a rotation, though, unlike a matrix, it does not represent a position or scale, or other matrix affects. It's advantage of a matrix is that it only has 4 components, instead of 16, like that of a matrix. It also doesn't suffer from gimbal lock, an affect that happens to a matrix over time where it's X, Y and Z axis become less and less perpendicular. Like a four-dimensional vertex, it's represented by an x, y, z, and w component. It's best, however, to avoid setting these directly, as they don't corespond to the same meanings as those of a V4. Instead, prefer the constructors listed in the Constructors section of this document. Like a matrix, order of multiplication matters.

Constructors

  • Quaternion() - Create a identity quaterion.
  • Quaternion( Quaternion ) - Create a quaternion that's a copy of another quaternion.
  • Quaternion( Matrix ) - Create a quaternion that's a copy of another rotation of a matrix.
  • Quaternion( V3, angle ) - Create a quaternion that's a rotation about an axis.
  • Quaternion( Quaternion, Quaternion, delta ) - Create a quaternion that's a Slerp (spherical-linear interpolation) between two quaternions, by a delta.
  • Quaternion( x, y, z, w ) - Create a quaternion from component values.
  • QuaternionZero() - Create a quatenrion full of 0s.
  • QuaternionRotationX( angle ) - Create a rotation about the x axis quaternion.
  • QuaternionRotationY( angle ) - Create a rotation about the y axis quaternion.
  • QuaternionRotationZ( angle ) - Create a rotation about the z axis quaternion.
  • QuaternionEuler( pry ) - Creates a Quaternion that from Euler coordinates.
  • QuaternionEuler( pitch, roll, yaw ) - Creates a Quaternion from Euler coordinates.
